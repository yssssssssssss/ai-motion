# 受控 AI 动效生成 Todo List

## 已完成基线

- [x] 首页预览懒加载与首屏自动加载，避免重资产组件首次空预览。
- [x] 缩略图预览增加循环播放兜底，支持 `motionReplay` fallback。
- [x] 重资产策略初版：非编辑目标的超大 `assets.css` 外置加载。
- [x] 组件健康度分析：可预览源码、动效检测、播放协议、可编辑参数、导出能力。
- [x] 生成就绪度分析：受控参数、图层画像、可替换图层、Plus 控制、规范 Skill 匹配。
- [x] 首页卡片展示健康度、生成就绪状态和图层数量。

## P0：生成前门禁

- [x] 定义显式 `DesignSpecSkill` 注册表，替代当前基于关键词的规范 Skill 推断。
- [x] 为组件 manifest 增加可选规范绑定字段，记录适用规范、禁用项、强制规则和置信度。
- [x] 为组件 manifest 增加显式 `layers` 字段，区分图片层、文案层、结构层、只读层和可替换层。
- [x] 在导入/上传流程生成图层清单，并要求用户确认可替换图层后才入库。
- [x] 增加生成门禁函数：没有规范绑定、没有受控参数、没有图层画像时禁止进入 AI 生成流程。

## P1：自然语言到受控变更

- [x] 设计 `GenerationPlan`：输入自然语言后只允许输出候选组件、规范 Skill、允许修改的参数、允许替换的图层和验收规则。
- [x] 实现候选组件 Top 2 检索与重排：同时考虑语义匹配、生成就绪度、规范匹配和 Plus 控制覆盖率。
- [x] 将 Plus 控制扩展到轨迹、节奏类参数，避免只覆盖速度、缓动和强度。
- [x] 增加参数归一化字典，把 `midX/endY/slideDistance` 等底层参数映射到稳定的自然语言控制概念。
- [x] 在生成前后做 diff 校验：AI 只能改白名单参数、白名单图层和允许的源码区域。

## P2：编辑体验

- [x] 首页增加生成就绪度筛选：全部、生成就绪、部分可生成、需补规范。
- [x] 组件详情中展示 readiness 诊断原因，明确告诉用户缺规范、缺图层还是缺 Plus 映射。
- [x] 编辑器中增加“图层替换”面板，和参数面板分离，避免图片替换混在普通参数里。
- [x] 上传弹窗增加入库检查清单：预览可播放、图层可识别、参数已确认、规范已选择。
- [x] 对“部分可生成”的组件提供补齐入口，例如补规范、声明图层、确认参数。

## P3：资产与预览可靠性

- [x] 给组件资产设预算：单组件入口 chunk、外置资产、内联 data URL 均需有阈值和测试。
- [x] 推动内置组件显式实现 `motionReplay`、`motionPause`、`motionSeek`，减少容器兜底。
- [x] 继续拆分重资产组件，避免首屏加载或测试环境读取大 base64。
- [x] 增加 Playwright 视觉回归：首屏卡片非空、缩略图循环、hover/点击动效可触发。

## P4：AI 生成闭环

- [x] 引入代码生成沙箱：生成新组件后先运行 schema 校验、源码白名单校验、预览校验，再允许导出。
- [x] 为视频、HTML/CSS、React、Next.js 上传分别定义最小转换协议。
- [x] 对生成结果建立评测集：规范符合度、动效可循环、图层可替换、导出可运行。
- [x] 增加失败回退策略：生成不合格时回到候选组件参数编辑，而不是输出不可控代码。
