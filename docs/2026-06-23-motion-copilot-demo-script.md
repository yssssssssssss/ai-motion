# Motion Copilot 发布演示脚本

日期：2026-06-23

范围：独立模块 `apps/motion-copilot` 与 `packages/motion-copilot-core`。

演示目标：用 5-8 分钟说明 Motion Copilot 不是旧编辑器里的附属功能，而是一套面向移动 App 动效设计的独立编排系统。重点展示真实图层、规范动效库、多图层时间轴、可编辑参数和导出能力。

## 演示前准备

- 启动页面：`pnpm --filter @motion-copilot/app dev`
- 访问地址：`http://127.0.0.1:5177`
- 建议浏览器宽度：桌面 1280px 以上。
- 准备素材：1 张前景图片、1 张背景图片。没有素材时也可以用内置示例完成演示。
- 演示前清理本地状态：进入页面后点击“清除自动保存”，再刷新页面。

## 开场讲法

Motion Copilot 当前阶段解决的不是“生成一段看起来会动的动画”，而是把移动 App 动效拆成 AI 和设计师都能理解的结构化对象：图层、规范动效、时间轴、开始时间、时长、串并行关系和导出结果。

当前版本的“生成动效”先使用本地 deterministic 结构化生成器，暂不接入自然语言大模型。原因是我们先把底层的可控编排能力搭好，避免做成不可编辑、不可交付的伪 AI 动画。

## 演示路径

### 1. 新建项目

操作：

1. 点击“开始使用”。
2. 点击“新建空白”。
3. 说明左侧是项目、图层素材和规范动效库；中间是实时画布；底部是多图层时间轴；右侧是参数面板。

讲解重点：

- 当前模块是独立系统，不复用旧编辑器主流程。
- 默认围绕“当前图层”工作，不再有“目标对象 / 主目标”切换。

### 2. 创建真实图层

操作：

1. 在左侧“动效描述”输入“做一个弹窗进场，标题先出现，按钮稍后轻弹”，点击“生成动效”。
2. 展示左侧“本次编排解释”，说明系统会解释每个片段为什么这样安排。
3. 展示底部时间轴生成的多图层编排片段，说明当前是本地结构化生成，不是大模型。
4. 再次点击“生成动效”，展示系统跳过重复同槽位片段并给出修正说明。
5. 在左侧“图层素材”点击“新增文本”。
6. 选择文本图层，在右侧修改文案、字号、颜色、对齐方式。
7. 点击“导入图片”添加前景图片，或在右侧对选中图片点击“上传图片”替换图片。
8. 在画布中拖拽图层位置，拖动边角调整图片尺寸。

讲解重点：

- 用户可以创建和编辑自己的图层，不再只能使用固定模板。
- 图片保持上传比例，并支持画布内拖拽、缩放和层级控制。
- 左侧图层列表支持选中、重命名、排序、复制、隐藏、锁定、删除。
- 本地结构化生成结果不是黑盒动画，用户能看到作用图层、时间轴片段、生成理由和修正说明。

### 3. 使用规范动效库

操作：

1. 选中文本或图片图层。
2. 展示左侧“App 规范动效”的分类 Tab：推荐、组合、转场、导航、横向、弹窗、底板、遮罩、卡片、反馈、加载、骨架、列表。
3. 在“当前图层推荐”里点击一个动效卡片，说明它会立即应用到当前图层。
4. 点击动效卡片右侧 `+`，把动效加入底部时间轴。
5. 使用搜索框搜索“进入”或“骨架”，展示跨分类查找。

讲解重点：

- 动效不是散乱模板，而是从移动 App 动效规范里提炼的原子能力。
- 每个动效都有默认时长、槽位、缓动类型和约束说明。
- 组合预设由多个原子动效组成，适合弹窗进场、底部面板、Tab 切换、加载转成功等常见场景。

### 4. 多图层时间轴编排

操作：

1. 给标题图层添加“进入屏幕”。
2. 给图片图层添加“容器变化”。
3. 给按钮或另一个文本图层添加“微反馈”或“横向切换”。
4. 在底部时间轴横向拖拽卡片，调整开始时间。
5. 将不同轨道上的卡片上下对齐，展示并行播放。
6. 拖动卡片到另一个图层轨道，展示动效作用图层变化。
7. 点击卡片，右侧面板展示该片段的开始时间、时长、串并行关系、缓动和起止状态参数。

讲解重点：

- 用户看到的是“开始时间 + 时长”，不是容易混淆的延迟字段。
- 上下对齐表示多图层同一时间发生动作。
- 错位表示串行或带时间差的编排。
- 底层仍保留兼容字段做串并行换算，但不暴露给用户。

### 5. 预览与循环

操作：

1. 点击“播放”。
2. 点击“循环”，展示动效循环播放。
3. 调整底部编排区域高度，说明预览区会响应缩放。

讲解重点：

- 预览不是静态 demo，时间轴和右侧参数调整会立即影响画布。
- 循环播放适合设计师反复观察节奏。

### 6. 导出

操作：

1. 在底部编排栏点击“下载编排 JSON”，展示结构化时间轴协议。
2. 点击“下载参数表 MD”，展示研发可读的动效参数表。
3. 点击“下载组合 HTML”，展示可独立预览的组合动效 HTML。
4. 点击右侧“规范”里的“下载完整 HTML”，展示完整页面级 HTML/CSS 导出。

讲解重点：

- JSON 编排协议包含画布、图层、轨道、片段、开始时间、结束时间、时长和缓动信息。
- JSON 导出服务系统对接和后续 AI 结构化生成，参数表服务研发沟通，HTML 导出服务预览验证。
- 下一阶段会继续增强 React/CSS 片段。

## 推荐演示组合

### 组合 A：弹窗进场

- 背景或遮罩：遮罩浮现
- 弹窗图片或面板：弹窗反馈
- 标题：进入屏幕
- 按钮：轻微反馈

讲法：这是移动 App 最常见的弹层打开路径，重点体现遮罩、内容、按钮不同层级之间的先后关系。

### 组合 B：加载转成功

- 骨架屏：骨架屏加载
- 内容图层：加载转成功
- 标题或按钮：进入屏幕

讲法：这体现规范约束，系统会避免骨架屏和位移类动效冲突。

### 组合 C：Tab 切换

- 指示器或标题：屏幕内移动
- 内容图层：横向切换

讲法：这体现多图层并行动效和时间轴对齐能力。

## 可能被挑战的问题

### 这是不是不够 AI？

回答：

当前版本没有把聊天框放在第一层，这是刻意选择。真正可交付的 AI 动效工具必须先有结构化底座：图层、规范动效、时间轴、参数、约束和导出。没有这些，AI 只能生成不可控动画；有了这些，AI 下一步才能生成可编辑、可验证、可交付的动效编排。

### 为什么不是直接接入大模型一句话生成？

回答：

现在已经有本地结构化生成入口，但暂不接大模型。后续大模型生成的目标也不是一段黑盒动画，而是一组结构化结果：选择哪些图层、使用哪些规范动效、开始时间是多少、哪些并行、时长和缓动参数是什么。生成结果会直接落到底部时间轴，用户还能继续编辑。

### 当前和原 8 大体系还有关系吗？

回答：

当前系统已经从“体系文档展示”转成“可操作的动效编排工具”。8 大体系更适合作为知识来源和约束框架，而不是界面上必须照搬的菜单结构。界面现在按移动 App 实际工作流组织：图层、规范动效、时间轴和参数。

### 为什么删除目标对象？

回答：

用户实际操作的是图层。保留目标对象会让概念重复：左侧有图层，右侧又有目标对象，时间轴还有轨道。现在统一为当前图层和图层轨道，理解成本更低。

## 当前边界

- 暂不接入自然语言大模型生成；当前“生成动效”为本地 deterministic 结构化生成。
- 暂不合并进旧编辑器主流程。
- 暂不复用旧系统能力。
- 导出能力目前支持 HTML/CSS 验证、编排 JSON 协议和研发参数表，React 片段待下一阶段增强。

## 演示验收标准

- 用户能看到真实图层创建、上传、编辑和拖拽。
- 用户能理解规范动效库不是模板堆叠，而是可组合原子动效。
- 用户能看到多图层时间轴的并行、串行和开始时间控制。
- 用户能通过循环预览观察节奏。
- 用户能理解当前 AI Native 路线：先结构化，再接入大模型生成。
